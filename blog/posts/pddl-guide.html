Deep Dive into PDDL: From Basics to Advanced Planning

February 10, 2025

Introduction

Planning Domain Definition Language (PDDL) is a standardized format for describing planning problems in artificial intelligence. This blog post provides a comprehensive guide to PDDL, starting from basic concepts and working our way up to implementing a custom planner and exploring advanced features like PDDL Stream. Whether you're new to automated planning or looking to deepen your understanding, this guide will help you master PDDL through practical examples and hands-on implementation.

1. Understanding PDDL Basics

PDDL separates the description of a planning domain from specific problem instances. This separation allows us to define generic action templates and predicates that can be reused across multiple problems. Let's explore this through a classic Blocks World example.

1.2 Code Breakdown

Domain Components:

  • Requirements: Specifies STRIPS and typing capabilities
  • Types: Defines the 'block' type for objects
  • Predicates: Describes possible relationships and states
  • Actions: Defines possible transitions in the world state

Problem Components:

  • Objects: Declares specific blocks in the world
  • Initial State: Describes the starting configuration
  • Goal State: Specifies the desired final configuration

3. Introduction to PDDL Stream

PDDL Stream extends traditional PDDL to handle continuous parameters and sampling-based planning. This is particularly useful for robotics and real-world planning problems where we need to deal with continuous quantities like positions and orientations.
