# Document retriever的讨论与实现
现在我们已经有了解析pdf的能力，也有了基本的BERT文本理解能力，想要完成一个能够工作的问答系统，还需要有一个document retriever，即文档搜索引擎。这个搜索引擎能够找到相对合适的文本文档，不至于让BERT处理过多的文本。BERT处理文本数量太多时，一方面速度会非常慢，另一方面它可能无法理解过于复杂的语义导致走索的精确度下降。

## 当我们想要retrieve的时候，我们想要找的是什么？
retriever的任务是找到“相关的”文档。这里的相关指的是什么呢？指的是语义的相关。也就是说我们希望找到“和我们的问题描述同一个领域的文档”。于是问题和文档的相似度就能够成为搜索的指标。

在大多数情况下，文本的相似就能够代表语义的相关。我们通常认为有着更多共同词汇的句子是相关的。所以我们可以使用cosine similarity来判定相关性。除了cosine similarity，还有一种技术叫做simhash，也可以用来分析处理不同文章和句子的相关性。一般来说，cosine similairty更适用于短的段落和句子，simhash更适用于长篇的文章。下面我们来分别更详细的介绍这两种技术。

### Cosine similarity
大家在小学二年级的时候都学过，判定两个向量的相似度，可以用他们的内积空间来判断。在欧几里得空间中，这个内积空间就是cos值。当他们的cos值为正时，值越小，说明它们的夹角越小，于是这两个向量就越接近。
$$sim(\vec{x}, \vec{y}) = \frac{\vec{x}\vec{y}}{||\vec{x}||\times||\vec{y}||}$$
如果我们将每一个文本看作一个向量，这个向量的内容是文本中包含的字符，那么我们也能够获得一个线性空间。在这个线性空间中可以使用cosine similarity考量两个文本的相似度。

但是单纯的使用这种办法有着极大的缺陷。首先，每个文档中都会出现很多常用词。在中文中这些词就是“的”、“是”之类的助词，在英文中就是"the", "a"之类的对于文章的核心含义没有影响的词。大量存在这些词不但会使向量维度极大的上升，导致计算量增大，也会使文章之间的相似度增加。这是因为这些词在每个文章中都会出现，频率甚至也相近，于是在这些词所代表的维度上，两个文本的向量也会非常相近。又因为这些词出现的频率肯定明显多于很多关键词，于是这种“无用的相似”就可能会掩盖真正需要我们辨别的相似。

所以我们使用nltk中的stopwords来去除这些常用词：
```python
from nltk.corpus import stopwords
from nltk import download
download('stopwords')
stop_words = stopwords.words('english')
data = row['text'].lower().split()
data = [w for w in data if w not in stop_words]
query = text.lower().split()
query = [w for w in query if w not in stop_words]
```
其中data是文章的数据，query是问题的数据。

但是单纯这样依然不够。cos sim还有两个最大的问题：
- 无法判断相似词汇
- 无法判断一词多义

我们来举两个例子："The man plays violin"和"The man plays badminton"。这两个句子语义差异还是蛮大的，一个是演奏小提琴，一个是打羽毛球。但是如果只从文本的角度来看的话，这两句话之间只差了一个单词，所以cosine similarity很有可能判定这两个句子非常相似。除此之外，"Hello world!"和"Hi world!"这两个短语语义几乎是完全相同的，但是它们在cosine similarity的眼中存在接近50%的差异，所以很有可能被认为是两个关系不大的短语。

为了解决这些问题，我们使用Soft Cosine Similarity。这是一种更加考虑语义的cosine similarity。我们在这里不对其原理进行过分详细的考察，直接[调库](https://radimrehurek.com/gensim_4.0.0/auto_examples/tutorials/run_scm.html#:~:text=Soft%20Cosine%20Measure%20(SCM)%20is%20a%20method%20that%20allows%20us,4%5D%20vector%20embeddings%20of%20words.)。

然后我们在retriever class中加入如下function：
```python
def cosineSimilarity(self, text):
        fasttext_model300 = api.load('fasttext-wiki-news-subwords-300')
        download('stopwords')
        stop_words = stopwords.words('english')
        maxSmi = 0.0
        maxDoc = ''
        selected = ''
        for _, row in self.doc_features.iterrows():
            print("Computing similarity of " + row['name'])
            data = row['text'].lower().split()
            data = [w for w in data if w not in stop_words]
            query = text.lower().split()
            query = [w for w in query if w not in stop_words]

            # prepare a dictionary and a corpus
            documents = [query, data]
            #  print(data)
            #  print(documents)
            dictionary = corpora.Dictionary(documents)

            # Convert the sentences into bag-of-words vectors
            query = dictionary.doc2bow(query)
            data = dictionary.doc2bow(data)
 
            # prepare the similarity matrix
            similarity_index = WordEmbeddingSimilarityIndex(fasttext_model300)
            similarity_matrix = SparseTermSimilarityMatrix(similarity_index, dictionary)
            
            # Compute soft cosine similarity
            similarity = similarity_matrix.inner_product(query, data, normalized=(True, True))
            if similarity > maxSmi:
                maxSmi = similarity
                maxDoc = row['name']
                selected = row['text']
            print(f"The similarity to {row['name']} is {similarity}")
        return maxDoc, selected
```

### simhash
为了解决上述的搜索速度慢的问题，我们现在尝试是用另一种方法来对比问题和文档的相似程度：simhash

SimHash是用来快速检验两个集合的相似度的。谷歌的Google Crawler就使用了这种方法寻找内容相似的页面。它将文本根据哈希表map到了另一个文本空间，然后进行对比。这个问答系统中使用的SimHash是基于[GitHub](https://github.com/1e0ng/simhash) 实现。所有的关于SimHash的代码都可以在这个仓库的simhash文件夹中找到。

把相应的文件下载下来，然后我们只需要在class中增加一个新函数即可：
```python
def simHash(self, text):
    distance = 1e5
    bestDoc= ''
    reText = ''
    for _, row in self.doc_features.iterrows():
        # print("Computing similarity of " + row['name'])
        sh = Simhash(row['text'])
        sh2 = Simhash(text)
        curDistance = sh.distance(sh2)
        if distance > curDistance:
            distance = curDistance
            bestDoc = row['name']
            reText = row['text']
        print(f"The distance to {row['name']} is {curDistance}")
    return bestDoc, reText
```
值得注意的是simHash算法中度量两个集合相似度的是distance，cosine similarity中度量的是similarity。一个是越小说明相似度越高，一个是越大说明相似度越高。自己实现的小伙伴不要搞混了。
