一个新的系列：问答系统

对于这个系列本身的介绍

现在网上已经有很多优质的问答系统的介绍和教程了。比如Losin同学在知乎上发的解开知识库问答KB-QA的面纱系列。但是我在中文社区中很少看到能够直接上手实现出来一个问答系统的文档和教程。只学习理论是非常枯燥的，这无疑对于新手来说十分劝退。

从我的个人经验来讲，学习计算机的过程就像是造船。我们虽然目标是造出一艘能够远航的巨轮，但如果一上来就打铁，一个螺丝一个螺丝的拧，很快就会因为枯燥乏味而放弃造船。但是如果我们先来一段木头，了解并熟悉一下浮力的性质，然后将木头优化一下——中间挖空，造出来一艘独木舟，然后再不断地对独木舟进行改进：把一根木头换成木板的拼接……以此类推，造船就能够给我们带来巨大的成就感，我们也会以此为动力不断地进步，直到我们造出来巨轮。如果遇到了好的机遇、老师或者伙伴，我们甚至能造出来航空母舰呢！

所以如果能一边写代码一遍学习理解问答系统，我相信是非常有益的。

这个系列就是旨在让学习者能够跟着一起做一个比较简单、但是有效的问答系统。我会从基本的语言模型的应用开始，逐步完善代码，最后实现一个可以工作的问答系统。然后我会演示如何将这个问答系统部署到服务器上。如果有时间的话，我可能也会实现一个安卓应用，展示如何在手机应用中加入问答系统。

源代码我会发布在GitHub上，有需要的同学可以自行下载。但我还是建议自己动手把所有的代码敲一遍，这样才能更好的理解整个系统的实现过程。

在这个系列中我参考了很多文章与博客，我会将自己参考的全部列下来。如果有任何遗漏，请通知我，我会加上。另外如果发现这个系列有错误或者不准确、不是最优的情况，欢迎指正。希望我们能够共同进步。

前置知识要求

这个系列的难度不会很高，是一个基础入门的级别，适合所有对问答系统有兴趣的计科、非计科同学学习阅读。但完全掌握理解本系列中的问答系统还是需要有一点前置知识。

如果没有基础的同学可以点进我提供的链接。这些都是我个人认为非常不错的学习资源。

  • Python programming
  • Linear Algebra
  • Machine Learning
  • Deep Learning

以上内容并不需要都精通以后再来做问答系统，我们可以在遇到不会的东西之后去上述提供的链接中找信息。

搭建环境

我强烈建议大家使用Python环境，而不是直接在自己的电脑上运行。如果直接在电脑上运行，可能会出现dependency紊乱的情况，各种包的调用相互影响。

首先安装miniconda,按照文档中给出的教程一步一步来即可。然后如果你使用windows，打开conda内部的conda powershell，如果你使用linux，则直接打开terminal。

然后运行代码

1
2

conda create -n QAsystem python=3.10
conda activate QAsystem

就进入conda环境了。然后根据你的电脑情况（有无GPU,CUDA的版本是什么）安装PyTorch. 然后我们的环境就安装好啦！来一起愉快的玩耍吧！

问答系统介绍

问答系统是一种人机交互的模式。它的流程如下：

人们向计算机提出问题 -> 计算机解析人类的自然语言 -> 计算机从数据库中搜索信息 -> 计算机回答问题

比如说我们向谷歌提问：幻影旅团的团长是谁？

谷歌就会告诉我们：是库洛洛·鲁西鲁！这并不是简单的搜索文档，而是谷歌所搜引擎解析了我们的提问以后，又解析了相关文档而给出的智能答案。我们想要做的问答系统就要有这种能力。

当然了，解析英文比中文简单不少，而且英文的自然语言模型有着原生的预训练，所以我们的这个模型使用英文实现的。感兴趣的同学可以试着用中文实现一下问答系统，欢迎pull request。

于是我们可以直观地看到，建立一个问答系统的难点主要有两个：

  • 如何让计算机理解人类提问所使用的自然语言
  • 如何让计算机寻找有效信息

处理人们的提问可以使用比较传统的语义分析，也可以用语言模型。我们在实现的时候会直接使用深度学习的模型，对于语义解析感兴趣的小伙伴可以去知乎看Losin的文章。

那计算机是如何找到有效信息的呢？问答模型在这里会分为两类：Open Domain Q&A system和Closed domain Q&A system。Open domain就是没有寻找信息的范围限制，可以从网络上直接搜寻海量的数据，找到合适的文档后进行解析并回答问题。Closed domain就是给定一些文档，可以是说明书，可以是财政汇报，也可以是规章制度文件，我们要从这些被指定的文件中提取想要的信息，回答提问者的问题。

无论是Open domain还是Closed domain，正常情况下都没有办法对所有文件进行自然语言处理，因为这样会耗费很高的算力。即使能处理，模型分析的大多数也是和我们问的问题关系不大的文档，所以我们想要先挑选和问题最相关的文档进行分析处理，这个挑选文档的部分被称为document retriever。现在我们有了挑出来的文档，就需要从里面找到更加具体的问题的答案了。为了找到这个答案，问答系统需要“理解”文档里在说什么，于是我们需要自然语言处理的模型来解析文档中的具体内容，这个解析文档的部分被称为document reader。

在这个系列中，我们就会一步一步的实现document retriever和document reader。
